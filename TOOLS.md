# Dev tools

The index of every development tool in this repo: what it does, where it
lives, and how to run it. In-game tools ship inside the debug layer;
generator scripts live under `tools/`; browser tools are built into the web
bundle next to the game and deployed with it (see the [tools index
page](https://potomak.github.io/VaniaVolpe/tools.html)).

## In-game (debug layer)

Toggle the debug layer with **D** in any build (desktop, terminal, web). A
red square in the top-left corner means it is active.

- **Hotspot & geometry overlay** — always on while debugging: cyan outlines
  for hotspots, magenta dots for POIs, translucent red shading over
  non-walkable cells of the scene's walk grid (`src/debug.c`).
- **Rect picker** — drag anywhere to print an `SDL_Rect` to the log; how
  hotspot and walkable rectangles are authored against the live scene.
- **Walk-mask paint mode** — press **W** (in a scene with a walk grid) to
  paint walkability over the live scene: **left-drag** marks cells walkable,
  **right-drag** blocks them, **S** saves
  `<assets>/common/<scene>/walkable.walk` into the source tree (native
  builds, run from the repo root). A yellow marker + screen frame show the
  mode is on; mouse input is swallowed so painting doesn't walk the actor.
  Scenes load a committed mask in `walk_grid_init` (falling back to their
  rect tables on any parse error); the `.walk` format is documented in
  `MOVEMENT.md`.

## Browser tools (deployed with the web build)

Built by `make web` into `build/web/` and deployed to GitHub Pages alongside
the game; the entry point is
[`tools.html`](https://potomak.github.io/VaniaVolpe/tools.html).

- **Asset catalog** — [`catalog.html`](https://potomak.github.io/VaniaVolpe/catalog.html):
  browses every adventure's assets per locale, plays audio and animations,
  and flags locales missing a file the reference locale (`it_IT`) has.
  Generated by `tools/gen_asset_catalog.py` (+ `src/emscripten/catalog.html`),
  which also emits `catalog.json` and copies the raw assets under
  `build/web/assets/`.
- **Walk-mask editor** — [`walk_editor.html`](https://potomak.github.io/VaniaVolpe/walk_editor.html):
  the browser counterpart of the in-game paint mode. Pick a scene background
  from the catalog, paint the 80×60 grid over it (left: walkable, right:
  blocked), load an existing committed mask or a local `.walk` file, and
  export a `walkable.walk` to commit. Useful when no native build is at hand.

## Generator scripts (`tools/`)

Run from the repo root; outputs are committed, so builds and CI never run
them.

- **`gen_playtest.py`** — turns a playthrough script
  (`test/scripts/<name>.json`, the single source of truth shared with the
  browser test) into a generated C header the native headless test drives.
  Run automatically by `make test`.
- **`gen_asset_catalog.py`** — emits the asset catalog (above). Run
  automatically by `make web`.
- **`gen_lipsync.py`** — runs Rhubarb Lip Sync over every dialogue WAV and
  writes the committed `.cues` / `.words` sidecars that drive the talking
  animation (see `SPEECH.md`). Needs the Rhubarb CLI on `PATH` or
  `$RHUBARB`; idempotent by mtime, `--force` regenerates.
- **`gen_asset_tasks.py`** — turns a per-adventure asset manifest
  (`src/adventures/<adv>/assets/tasks.json`) into
  [`ASSETS_TODO.md`](ASSETS_TODO.md): a grouped, checkbox to-do list of every
  animation, voice line and still image still to author, each with a GitHub
  **upload** link that lands in the right directory for the raw files, and a
  status read live from the repo (frames uploaded / WAV recorded / image
  uploaded). Re-run after an upload to refresh the boxes. Stdlib only.
- **`gen_en_us_placeholders.sh`** — scaffolds a complete `en_US` locale from
  `it_IT` (silent WAVs, copied images) so the strict locale lookup never
  misses while real translations are pending.
- **`gen_depth_variants.py`** — emits placeholder far/mid depth-variant
  sprites (see `DEPTH_AND_CAMERA.md` Phase 2): each frame of a sprite sheet
  downscaled by `--scale` (nearest-neighbour), repacked vertically with a
  matching `.anim`. Placeholders to develop depth bands against; real art
  replaces them file-for-file. Needs Pillow.

## Test harnesses

- **Headless playthrough** — `make test` → `./vaniavolpe_test`: offscreen
  renderer + dummy audio, drives the scripted playthroughs and the unit
  tests (`test/`), asserting dialogue order and geometry invariants. The CI
  gate.
- **Browser playthrough** — `test/web/run_playtest.js`: Puppeteer drives the
  deployed-shape web build through the same JSON script, saving screenshots.
  Run by `.github/workflows/web-test.yml`.
- **Terminal build** — `make terminal` → `./vaniavolpe_terminal`: the whole
  game as libcaca ASCII art; handy for quick play-testing over SSH.

## Adding a tool

Keep this file the index: one entry per tool, a sentence on what it's for,
and where it runs. Browser tools get a card in `src/emscripten/tools.html`
too, and a `cp` line in the Makefile's `web` target.
